# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities privately. Open the repository's
**Security** tab and choose **Report a vulnerability** to file a private report
through GitHub
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

Do not open a public issue for security problems.

## Security Best Practices for Students

When using these course materials:

### DO ✅
- Use **Azure free trials** or **student credits** only
- Store credentials in **Azure Key Vault** or **environment variables**
- Follow the **governance labs** (Module 1) before deploying resources
- Enable **budgets and alerts** to prevent unexpected charges
- Use **resource locks** on production-like environments
- **Delete resources** after completing labs to avoid costs

### DON'T ❌
- Commit Azure credentials, subscription IDs, or tenant IDs to Git
- Share your Azure subscription ID publicly
- Use production Azure subscriptions for learning exercises
- Leave resources running after completing labs
- Store passwords or API keys in scripts or configuration files
- Disable Azure security features for convenience

Thank you for helping keep SkyCraft secure and educational! 🛡️
